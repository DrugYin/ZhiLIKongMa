# 智力控码软件 设计说明书

著作权人名称：李崇伟

## 一、软件概述

### 1.1 软件简介

智力控码软件是一套面向教育培训场景的训练管理系统，由微信小程序端、CloudBase 云函数后端、CloudBase 文档数据库/云存储和 Vue 后台管理网站组成，用于完成训练项目配置、班级组织、任务发布、作业提交、审核评分、积分排行、公告通知和运营管理。

### 1.2 开发背景

在编程、无人机、机器人等训练课程中，教师需要发布不同难度和类别的训练任务，学生需要在线提交作品，运营人员需要查看训练数据并管理系统参数。传统线下记录方式难以追踪提交过程、审核结果和积分激励，因此设计本系统以统一承载教学训练和运营管理流程。

### 1.3 设计目标

- 为学生提供清晰的任务中心、提交入口、提交记录和排行反馈。
- 为教师提供班级、任务、审核和学生训练进度管理能力。
- 为管理员提供配置、项目、用户、班级、任务、提交、排行、公告和日志管理能力。
- 通过 CloudBase 云开发降低部署复杂度，提升数据访问和文件上传的稳定性。
- 建立统一权限校验、参数校验和操作日志机制，保证业务数据可追踪。

## 二、需求分析

### 2.1 功能需求

1. 微信登录、注册资料完善、角色切换与用户信息维护。
2. 学生端支持加入班级、查看训练任务、提交作业、查看提交记录和排行榜。
3. 教师端支持创建和管理班级、发布训练任务、查看学生提交并进行审核评分。
4. 任务体系支持公开任务和班级任务，包含项目分类、难度、积分、截止时间和素材附件。
5. 审核体系支持入班审批、作业审核、反馈记录、反馈附件、积分发放和操作追踪。
6. 排行榜体系支持周榜、月榜、总榜以及后台排行数据查询。
7. 后台管理网站支持运营总览、系统配置、项目配置、用户、班级、任务、提交、公告和日志管理。
8. 基于腾讯云开发 CloudBase，使用云函数、文档数据库和云存储承载后端能力。

### 2.2 性能需求

- 小程序页面初始化时按需加载用户、项目、班级和任务数据，减少无效请求。
- 云函数查询默认分页，列表接口限制单次读取数量，避免数据库压力过大。
- 图片和文件上传前限制数量、大小和类型，降低云存储和网络传输压力。
- 后台管理列表支持筛选、分页和状态查询，适配运营人员高频检索。

### 2.3 安全需求

- 小程序端通过微信身份获得 openid，并在云函数中依据 openid 查询用户权限。
- 教师类操作校验用户是否具备教师角色，并校验班级归属或任务管理权限。
- 后台管理端通过 CloudBase Web Auth 与 `users.admin_auth_uid` 绑定管理员身份。
- 所有后台写操作通过 `admin-*` 云函数执行，并记录操作日志。
- 任务、提交、班级和用户等输入参数在云函数中进行规范化和校验。

### 2.4 用户需求

- 学生希望快速查看任务、提交成果、了解审核状态和积分排名。
- 教师希望便捷发布任务、查看提交、审核反馈并管理班级成员。
- 管理员希望集中查看运营数据、维护参数和追踪关键操作。

## 三、总体设计

### 3.1 系统架构

```text
微信小程序端
  | wx.cloud.callFunction / wx.cloud.uploadFile
  v
CloudBase 云函数
  | 权限校验、参数校验、业务处理、操作日志
  v
CloudBase 文档数据库 / 云存储

后台管理网站 Vue 3
  | CloudBase Web SDK / Web Auth
  v
admin-* 管理云函数
  | 管理员校验、审计、分页查询、配置更新
  v
CloudBase 文档数据库 / 云存储
```

### 3.2 模块划分

- 小程序应用模块：`miniprogram/app.js`、`app.json` 负责全局初始化、页面注册和云开发初始化。
- 页面模块：学生端、教师端、登录页、公告页等页面按业务目录拆分。
- 组件模块：公告面板、自定义导航栏、空状态、表单、加载态等公共组件。
- 服务模块：`services/api.js` 封装云函数调用，`auth.js`、`class.js`、`task.js` 等封装业务服务。
- 工具模块：常量、格式化、文件资源、截止时间、Toast、验证等通用能力。
- 云函数模块：用户、班级、任务、提交、审核、排行、公告和后台管理函数。
- 共享云函数模块：认证、权限、响应格式、操作日志、通知和任务访问判断。
- 后台管理模块：Vue 页面、路由、状态管理、CloudBase API 封装和主题样式。

### 3.3 技术选型

| 层级 | 技术 | 用途 |
|------|------|------|
| 小程序端 | 微信小程序原生框架、JavaScript、WXML、WXSS | 学生端和教师端业务界面 |
| 小程序 UI | TDesign Mini | 表单、列表、弹窗、上传等组件能力 |
| 后台管理端 | Vue 3、Vite、Pinia、Vue Router | Web 管理控制台 |
| 后台 UI | TDesign Vue Next、ECharts | 管理端组件和运营图表 |
| 后端 | CloudBase 云函数、wx-server-sdk、@cloudbase/node-sdk | 业务接口、权限校验和管理操作 |
| 数据 | CloudBase 文档数据库、云存储 | 用户、班级、任务、提交和文件数据 |

### 3.4 运行环境

- 小程序端运行在微信客户端和微信小程序基础库 3.0+。
- 云函数运行在腾讯云开发 CloudBase Node.js 运行环境。
- 后台管理端运行在现代浏览器，可部署至 CloudBase 静态网站托管。

## 四、详细设计

### 4.1 用户与认证模块

用户通过 `login` 云函数获取 openid，通过 `register` 写入用户资料。小程序端 `AuthService` 负责登录、注册、获取本地用户信息、角色切换和退出登录。用户数据存储在 `users` 集合，字段包含基础资料、角色列表、当前角色、积分、状态和管理员绑定信息。

### 4.2 班级管理模块

班级模块包括创建班级、编辑班级、删除班级、查询班级列表、获取班级详情、处理入班申请和班级成员管理。学生可通过班级码申请加入班级，教师在审核页面处理申请。系统使用 `classes`、`class_memberships`、`class_join_applications` 等集合维护班级关系。

### 4.3 任务管理模块

任务模块支持公开任务和班级任务。教师在任务编辑页填写标题、描述、项目、分类、难度、积分、状态、截止日期和素材附件。云函数对任务类型、可见性、难度、积分、截止时间和附件进行规范化处理，任务数据存储在 `tasks` 集合。

### 4.4 提交与审核模块

学生在任务详情或提交页上传作业，系统记录提交说明、图片、附件、状态和提交时间。教师审核后写入审核结果、反馈、附件和积分发放信息。提交数据存储在 `submissions` 集合，审核逻辑由 `review-submission` 处理。

### 4.5 排行榜模块

排行榜模块基于用户积分、任务完成情况和快照数据展示周榜、月榜、总榜等视图。`get-ranking` 提供小程序端查询，`refresh-ranking-snapshots` 用于刷新排行快照，后台管理端可查看排行数据。

### 4.6 后台管理模块

后台管理网站使用 CloudBase Web SDK 登录并调用管理端云函数。所有 `admin-*` 云函数先校验管理员身份，再执行查询或写操作。模块包括运营概览、配置、项目、用户、班级、任务、提交、排行、公告和日志。

## 五、数据结构设计

### 5.1 主要集合

- `users`
- `classes`
- `class_memberships`
- `class_join_applications`
- `tasks`
- `submissions`
- `projects`
- `ranking_snapshots`
- `system_config`
- `announcements`
- `announcement_reads`
- `operation_logs`

### 5.2 关键数据关系

- `users` 与 `class_memberships` 通过 openid 建立学生和班级关系。
- `classes` 与 `tasks` 通过 `class_id` 关联班级任务。
- `tasks` 与 `submissions` 通过 `task_id` 关联学生提交记录。
- `projects` 为任务提供项目、分类、难度、默认积分等配置。
- `operation_logs` 记录后台配置、项目、用户、班级、任务、公告等关键操作。

### 5.3 数据格式

系统主要使用 JSON 文档格式存储业务数据。时间字段使用 Date 或标准日期字符串；图片和附件使用 CloudBase fileID、文件名、文件大小等字段描述；状态字段采用枚举值，如 `draft`、`published`、`closed`、`pending`、`approved`、`rejected`。

## 六、接口设计

### 6.1 小程序端接口

小程序统一通过 `services/api.js` 中的 `callFunction` 调用云函数，返回结构包含 `success`、`message`、`data` 和 `error_code`。业务服务进一步封装为用户、班级、任务、排行、公告等 API。

### 6.2 云函数接口

当前主要云函数包括：admin-auth-check, admin-get-statistics, admin-manage-announcements, admin-manage-classes, admin-manage-config, admin-manage-logs, admin-manage-projects, admin-manage-rankings, admin-manage-submissions, admin-manage-tasks, admin-manage-users, create-class, create-task, delete-class, delete-task, get-announcements, get-class-applications, get-class-detail, get-class-invite-info, get-class-members, get-classes, get-my-class-status, get-projects, get-ranking, get-submissions, get-task-detail, get-tasks, get-user-info, get-user-roles, handle-join-application, join-class, login, refresh-ranking-snapshots, register, remove-member, review-submission, submit-task, switch-role, update-class, update-task, update-user。

### 6.3 后台管理接口

后台管理端通过 `admin-web/src/api` 下的接口模块调用 `admin-*` 云函数，各管理函数进行管理员校验、参数校验、分页查询和审计日志记录。

## 七、算法设计

### 7.1 权限判断算法

1. 获取调用者 openid 或 Web Auth uid。
2. 查询 `users` 集合中的用户记录。
3. 判断用户角色是否包含目标权限，如 `teacher` 或 `admin`。
4. 对班级、任务、提交等资源继续校验归属关系或可见性。
5. 权限通过后执行操作，否则返回 401、403 或业务错误码。

### 7.2 截止时间校验算法

任务截止时间由日期和时间字段组合，云函数将其解析为本地 Date 对象，校验格式合法性、日期越界和时间有效性。学生提交时结合任务状态和截止时间判断是否可提交，必要时结合超时规则处理积分。

### 7.3 上传资源处理算法

上传前根据配置限制图片和文件数量、大小与类型。上传后保存 fileID、文件名和文件大小，展示时通过文件资源工具转换为可预览地址，保证任务素材、学生提交和教师反馈可以统一处理。

## 八、界面设计

### 8.1 小程序界面

小程序采用学生端和教师端分区设计，底部自定义 TabBar 根据角色展示首页、排行榜、任务、审核、我的等入口。页面使用卡片、列表、表单、状态标签、上传控件和反馈弹层组织信息。

### 8.2 后台管理界面

后台管理网站采用左侧导航加内容区布局，包含运营概览、配置、项目、用户、班级、任务、提交、排行、公告、日志等页面。列表页面支持筛选、分页和状态展示，编辑页面通过表单或弹窗完成操作。

## 九、安全设计

- 身份校验前置到云函数，前端只负责发起请求和展示结果。
- 管理端写操作全部记录操作日志，便于追溯。
- 后台前端不直接修改敏感集合，统一通过管理云函数处理。
- 文件上传限制数量、大小、类型，降低异常资源风险。
- 对任务、班级、提交等关键参数做 normalize 和合法性校验。
